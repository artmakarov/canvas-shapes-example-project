# Canvas Shapes - JavaScript Canvas Application

Это JavaScript приложение на основе HTML5 Canvas для создания и управления фигурами с автоматическими соединениями. 

Проект разработан в рамках тестового задания на вакансию (native) JavaScript разработчика, с применением ООП, паттернов проектирования и соблюдением принципов SOLID.

Проект демонстрирует отличную реализацию современных подходов к разработке на чистом JavaScript без использования фреймворков.

Демо:
[artmakarov.github.io/canvas-shapes-example-project](https://artmakarov.github.io/canvas-shapes-example-project/)

## Ключевые технологии

- **HTML5 Canvas API** - Основная технология рендеринга
- **Vanilla JavaScript ES6+** - Чистая JS реализация без фреймворков
- **JSDoc типизация** - Типы и интерфейсы в комментариях
- **CSS3** - Стилизация интерфейса

## Архитектурные паттерны

### 1. Паттерн Шаблонный метод
- Скелет алгоритма в базовом классе, некоторые методы переопределяются в наследниках
- Все компоненты имеют четкие интерфейсы в `/js/interfaces/`
- Четкое разделение ответственности между рендерингом, логикой и UI
- Интерфейсы обеспечивают гибкость и тестируемость

### 2. Паттерн Абстрактная фабрика
- Фабрика семейств связанных объектов без привязки к конкретным классам
- Централизованное создание объектов с едиными правилами
- `ShapeFactory` и `ConnectionFactory` для создания объектов
- Легко расширяется новыми типами фигур или стилями соединений

### 3. Паттерн Менеджер
- `ShapeManager`, `ButtonManager`, `UIManager` для управления компонентами
- Чистое разделение ответственности
- Каждый менеджер отвечает за свою область

### 4. Паттерн Стратегия
- Легкая замена алгоритмов без изменения основного кода
- Разные генераторы соединений (`SmartConnectionGenerator`)
- Разные рендереры (`Canvas2DRenderer`)

## Основные компоненты

### Приложение (app/)
- `CanvasApp` - основное приложение с базовыми функциями
- `CanvasAppBase` - базовый класс для расширения
- `CanvasAppExtended` - расширенная версия с дополнительными функциями

### Фигуры и модели (models/)
- `Shape` (интерфейс) - базовый класс для всех фигур
- `Rectangle` - реализация прямоугольника
- `Connection` (интерфейс) - базовый класс для соединений
- `ArrowConnection` - стрелочные соединения между фигурами

### Фабрики (factories/)
- `ShapeFactory` - создание фигур
- `ConnectionFactory` - создание соединений
- Централизованное управление созданием объектов

### Менеджеры (managers/)
- `ShapeManager` - жизненный цикл фигур
- `ShapeManagerExtended` - расширенный менеджер с дополнительными функциями
- `ButtonManager` - управление UI кнопками
- `UIManager` - обновление состояния интерфейса

### Рендеринг (renderers/)
- `CanvasRenderer` (интерфейс) - базовый класс для рендерера
- `Canvas2DRenderer` - конкретная реализация с 2D Canvas API

### Генераторы (generators/)
- `SmartConnectionGenerator` - умное соединение между фигурами
- Выбор оптимальных сторон для подключения
- Минимизация длины линий

### Интерфейсы (interfaces/)
- Полный набор TypeScript-подобных интерфейсов
- Обеспечивают типобезопасность и документацию
- Легко расширяемая система типов

## Функциональность

- **Создание фигур** - Добавление новых фигур на холст
- **Выбор фигур** - Клик для выбора и выделения фигур
- **Автоматическое соединение** - Умное соединение между фигурами
- **Удаление фигур** - Удаление выбранных фигур
- **Очистка холста** - Полная очистка всех фигур
- **Адаптивный рендеринг** - Подстройка под размер окна

## Особенности реализации

### Гибкая система кнопок
- Легко добавлять новые функции без изменения основных классов
- Конфигурация кнопок через объекты
- Автоматическое управление состоянием кнопок

### Умный генератор соединений
- Выбирает оптимальные стороны для подключения
- Минимизирует длину линий

### Полная типизация
- Все методы и свойства имеют JSDoc аннотации
- Интерфейсы для всех основных компонентов
- TypeScript-подобные типы в комментариях

### SOLID принципы
- **S**ingle Responsibility: Каждый класс имеет одну четкую цель
- **O**pen/Closed: Легко расширять без изменения существующего кода
- **L**iskov Substitution: Подклассы могут заменять родительские классы
- **I**nterface Segregation: Чистые, сфокусированные интерфейсы
- **D**ependency Inversion: Зависимости внедряются через конструктор

### Расширяемость
- Легко добавлять новые типы фигур
- Простое добавление новых типов соединений
- Гибкая система UI без изменения основных классов

## Два варианта приложения

### 1. index.html - Базовая версия
- Основные функции: Add, Delete, Clear
- Автоматическое соединение фигур
- Выбор и визуальная обратная связь

### 2. example-extended-buttons.html - Расширенная версия
- Все функции базовой версии

## Запуск приложения

Это чистый HTML/JS проект без инструментов сборки:
- Откройте `index.html` или `example-extended-buttons.html` напрямую в браузере
- Компиляция не требуется

## Требования к браузеру

- Современные браузеры с поддержкой Canvas API
- JavaScript ES6+ функции
- Полифиллы не требуются для базовой функциональности

## Стиль кода

Проект следует строгим правилам кодирования:

- **Соглашения об именовании**: PascalCase для классов, camelCase для методов
- **Приватные члены**: с префиксом подчеркивания (`_shapes`, `_selectedShape`)
- **JSDoc аннотации**: Все методы и свойства имеют типизацию
- **SOLID принципы**: Строгое следование принципам объектно-ориентированного программирования
